Se realiza un commit y el primer push usando la linea de comandos. Algunos comandos utiles son:

  • git remote - ver los remotos del repositorio
  • git remote -v - ver los remotos del repositorio con sus URLs
  • git push <origin> <branch-name> (Nota: se solicitar√° username y password)

Cambio realizado desde la web de github con la finalidad de hacer un pull con el comando "git pull <remote> <branch-name>" (Nota: tengo cambios sin hacer commit, no se que vaya a pasar xD)

Cuando hay modificaciones de archivos que ya existian en el commit previo, se puede ejecutar directamente el comando "$git commit -am <mensaje>", que lo que hace es poner en staged los archivos modificados y enviarlos al commit con un solo comando (la 'm' se puede omitir si queremos que se abra el editor de texto para editar el mensaje).

Usando ramas

  • $ git branch - lista las ramas del repositorio
  • $ git branch <branch-name> - crear una nueva rama
  • $ git checkout <branch-name> - cambiar de rama
  • $ git checkout -b <branch-name> - crea y cambiar de rama
  • NOTA: tenemos que hacer push especificando las ramas para que se suban a github (claro, si eso es lo que queremos)
